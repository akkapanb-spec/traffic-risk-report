# แผน Sync ข้อมูลระหว่างช่วงเปลี่ยนระบบ (GAS → Supabase)

ปัญหา: ระหว่างที่กำลัง migrate ระบบเดิม (GAS + Google Sheets) ยังเปิดใช้งานอยู่
อาจมีประชาชนแจ้งจุดเสี่ยงใหม่ หรือเจ้าหน้าที่ลงข้อมูลเพิ่มใน Sheet
→ ข้อมูลใน Supabase จะตกหล่นถ้าไม่ sync ก่อนตัดระบบ

## หลักการ: Import ครั้งแรก → เปิดเว็บใหม่ → Sync ส่วนต่าง → ปิดทางเข้าระบบเดิม

### ขั้นที่ 1 — Import รอบแรก (ทำได้เลย)
- Import `data/risk_points_clean.csv` และ CSV ชีทอื่นๆ เข้า Supabase
- **จดวันเวลาที่ export ไว้** เช่น "export เมื่อ 18/07/2026 14:00"

### ขั้นที่ 2 — พัฒนา + ทดสอบเว็บใหม่
- ระบบเดิมยังใช้งานตามปกติ ข้อมูลใหม่เข้า Google Sheet ไปก่อน ไม่เป็นไร

### ✅ มีเครื่องมือช่วยแล้ว: `sync.html`

เปิด `https://akkapanb-spec.github.io/traffic-risk-report/sync.html`
กดปุ่ม "ตรวจสอบข้อมูลใหม่จากระบบเดิม" → ระบบจะดึงข้อมูลล่าสุดจาก Google Sheet
(ผ่าน GAS เดิม) มาเทียบกับ Supabase อัตโนมัติ แล้วสร้าง SQL เฉพาะส่วนต่างให้:
- จุดเสี่ยงที่แจ้งใหม่ในระบบเดิม
- สถานะที่เจ้าหน้าที่แก้ในชีต
- ข้อมูลการดำเนินการใหม่

กดคัดลอก → วางใน Supabase SQL Editor → Run — ทำซ้ำได้บ่อยเท่าที่ต้องการ
(ครอบคลุมระบบจุดเสี่ยงของประชาชน ส่วนชีทอุบัติเหตุ/บาดเจ็บ/เสียชีวิตของเจ้าหน้าที่
ยังใช้วิธี export CSV เป็นระยะตามเดิม เพราะ GAS เดิมไม่มี API ดึงข้อมูลชุดนั้น)

### ขั้นที่ 3 — วัน Cutover (วันตัดระบบ)
1. Export ชีทที่มีการเขียนข้อมูล ใหม่อีกรอบ:
   - `ข้อมูลจุดเสี่ยง` (ประชาชนแจ้งเพิ่ม + เจ้าหน้าที่อัปเดตสถานะ)
   - `ข้อมูลการดำเนินการ`
   - ชีทอุบัติเหตุ/บาดเจ็บ/เสียชีวิต ถ้าเจ้าหน้าที่ลงเพิ่ม
2. หา **แถวที่ใหม่กว่าวันที่ export รอบแรก** (ดูจากคอลัมน์วันที่)
   - ชีทเป็น append-only อยู่แล้ว → แถวใหม่จะต่อท้ายเสมอ นับแถวเทียบกับรอบแรกก็ได้
3. แปลง + import เฉพาะแถวใหม่เข้า Supabase (ส่งไฟล์มาให้ Claude แปลงให้ได้)
4. ⚠️ เช็คด้วยว่าแถวเดิมมี **การแก้สถานะ** ไหม (เช่น ยังไม่ได้ดำเนินการ → แก้ไขแล้ว)
   - ถ้ามี ให้อัปเดต status ในตาราง `risk_points` ให้ตรง (แก้ผ่าน Table Editor ได้ ข้อมูลไม่กี่แถว)
5. รูปที่อัปโหลดใหม่ในช่วงนี้ (อยู่ใน Drive) → ย้ายไป Cloudinary + อัปเดต URL

### ขั้นที่ 4 — ปิดทางเข้าระบบเดิม
แก้ GAS `doGet` ให้ redirect ไปเว็บใหม่ (ลิงก์เก่าที่แชร์ไปแล้วจะยังใช้ได้):

```javascript
function doGet(e) {
  return HtmlService.createHtmlOutput(
    '<script>window.top.location.href="https://<username>.github.io/traffic-risk-report/";</script>' +
    '<p>ระบบย้ายไปที่ใหม่แล้ว กำลังพาไป... <a href="https://<username>.github.io/traffic-risk-report/">คลิกที่นี่</a></p>'
  );
}
```

จากนั้น Deploy → Manage deployments → Edit → New version
ตั้งแต่นี้ข้อมูลใหม่จะเข้า Supabase ทางเดียว ไม่ต้อง sync อีก

### หมายเหตุ
- ระบบ "บันทึกข้อมูลอุบัติเหตุ สำหรับเจ้าหน้าที่" กับ "โครงการหมวกนิรภัย" เป็น GAS คนละตัว
  ยังเปิดใช้ต่อได้ (ปุ่มในเว็บใหม่ยังลิงก์ไปที่เดิม) — จะ migrate ทีหลังก็ได้
- ถ้าเจ้าหน้าที่ยังลงข้อมูลอุบัติเหตุผ่าน GAS ตัวนั้นลง Sheet เดิมอยู่
  ตาราง accidents/injuries/deaths ใน Supabase ต้อง sync เป็นระยะ (export → import แถวใหม่)
  จนกว่าจะ migrate ระบบเจ้าหน้าที่ด้วย
